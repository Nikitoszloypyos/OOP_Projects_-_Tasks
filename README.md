# Структура проекта и роль директорий


#### Проект работает по схеме: **UI → application → domain**, а **infrastructure** подключается через порты.

---
## Описание основных слоёв:

### src/
- Точка входа коду приложения.

### 1. src/domain/
- **Роль:** чистая бизнес-логика без I/O (только данные и правила).

### 2. src/application/
- **Роль:** сценарии приложения (оркестрация I/O и доменной логики), контракты к инфраструктуре.

### 3. src/infrastructure/
- **Роль:** детали реализации портов (БД/HTTP и пр.). Меняется без влияния на UI/домен.

### 3. src/UI/
- **Роль:** представление (React), роуты, компоненты, стили.

### 3. src/tests/
- **Роль:** тесты.


---
## Описание директорий каждого слоя:

## 1. domain:

### src/domain/entities/
- **Роль:** доменные сущности и их инварианты.

### src/domain/errors/
- **Роль:** доменные ошибки.

### src/domain/services/
- **Роль:** чистые алгоритмы/сервисы, не принадлежащие одной сущности.

### src/domain/value-objects/
- **Роль:** value-объекты с правилами.

---

## 2. application:


### src/application/ports/
- **Роль:** интерфейсы (контракты) для внешних зависимостей.

### src/application/use-cases/
- **Роль:** функции-сценарии, которые используют `ports` и домен.

---

## 3. infrastructure:

### src/infrastructure/DB/
- **Роль:** инициализация и схема локальной БД.

### src/infrastructure/HTTP/
- **Роль:** общий HTTP-клиент/обёртка.

### src/infrastructure/repos/
- **Роль:** реализации репозиториев (портов) поверх DB/HTTP.

---

## 3. UI:

### src/UI/assets/
- **Роль:** статические ресурсы.
- **Что лежит:** иконки, шрифты, JSON, и т.д.

#### src/UI/assets/images/
- **Роль:** изображения для интерфейса.
- **Что лежит:** PNG/SVG/JPG.

### src/UI/components/
- **Роль:** переиспользуемые UI-компоненты (без бизнес-логики).
- **Что лежит:** `Button.tsx`, `Card.tsx`, `Modal.tsx`, и т.п.

### src/UI/hooks/
- **Роль:** UI-хуки, возможно будет что то ещё

### src/UI/pages/
- **Роль:** страницы/роуты, композиция компонентов и хуков.

### src/UI/shared/
- **Роль:** вспомогательные вещи уровня UI (утилиты форматирования, типы).

#### src/UI/shared/@types/
- **Роль:** утилитарные типы UI-слоя (не доменные).

#### src/UI/shared/lib/
- **Роль:** UI-утилиты (форматтеры, i18n-хелперы), без React-зависимости по возможности.

### src/UI/styles/
- **Роль:** глобальные стили, токены дизайна, reset/normalize.
- **Что лежит:** `index.css`, `vars.css`, (или Tailwind конфиги).

#### src/UI/styles/themes/
- **Роль:** темы/палитры/дизайн-токены.
- **Что лежит:** `light.css`, `dark.css` или `light.ts`, `dark.ts`.

---
## Как работать с проектом:

### Установка зависимостей:

1. ```nvm install``` - установка версии node.js использующейся в проекте.

2. ```corepack enable``` - подтянет нужный пакет менеджер

3. ```npm install``` - установка всех зависимостей проекта.


### Для запуска и сборки проекта есть 3 команды:

- ```npm run dev``` — разрабатывать

- ```npm run build``` — собрать прод

-  ```npm run preview ```— посмотреть, как этот прод будет вести себя на сервере
